# 家庭投资投委会

## 概述

这个 skill 用来运行一个精简版“家庭投资投委会系统”：前面由 `5` 个独立专业角色 agent 分别分析，最后由 `1` 个“芒格思维总结 agent”负责综合结论。

这个 skill 的目标不是制造确定感，而是提升决策质量、暴露隐藏风险，并把模糊的投资冲动变成更克制、更可执行的下一步。

## 何时使用

以下情况适合使用这个 skill：

- 用户在问是否应该买入、加仓、减仓、再平衡或暂停投资
- 问题核心是家庭资产配置，而不只是市场观点
- 用户希望先看多个独立视角，再看综合结论
- 用户在做比较重要的投资动作，希望有一套稳定、可复用的分析流程
- 用户希望按月给 `标普500`、`纳指100`、`A股低波红利`、`黄金` 做战术层打分，并返回本月应投资的百分比
- 用户希望按月给 `标普500`、`纳指100`、`A股低波红利（512890）`、`黄金` 做战术层打分，并返回本月应投资的百分比

以下情况不适合使用这个 skill：

- 只是要一个简短的市场资讯摘要
- 只是做单一标的研究，且不涉及家庭约束
- 用户只想要一个单视角答案，不想走投委会流程

## 编制说明

为适配最大并发 `6` 个 agent，这个 skill 固定采用 `5 + 1` 结构：

- `5` 个专业分析 agent
- `1` 个芒格思维总结 agent

## 必要输入

以 [input-template.md](references/input-template.md) 中的家庭投资案例模板为标准输入。

如果用户信息不完整：

- 明确指出哪些缺失信息会实质影响判断
- 仍然基于现有事实给出最佳分析
- 不要默认用户是激进投资者、高收入家庭或能承受深度回撤

## 角色集合

角色提示词目录索引放在 [roles.md](references/roles.md)。

真正的角色提示词必须拆分存放在 `references/roles/` 下的独立文件中，每个角色一个文件。

禁止把多个角色重新拼回单个提示词文件再使用。

固定角色如下：

1. `family-cfo`
2. `family-chief-allocation-officer`
3. `family-risk-officer`
4. `macro-scenario-analyst`
5. `execution-strategist`
6. `munger-synthesis-chair`

这 5 个专业角色覆盖：

- 家庭现金流与资金期限
- 长期资产配置
- 风险与流动性
- 宏观情景
- 执行落地

“芒格思维总结 agent”负责在这些专业结论之上做最终收口，重点强调少犯错、机会成本和安全边际。

## 外部信息采集要求

在任何专业角色开始分析之前，`munger-synthesis-chair` 必须先完成一轮会前信息收集，并整理成统一资料包发给全部 `5` 个专业角色。

除非用户明确说不需要，否则以下来源属于默认必查项：

- 瑞银 `UBS` 关于标普500、纳指100、半导体等相关主题的最新研究或观点
- 摩根 `J.P. Morgan` 或 Morgan Stanley 关于标普500、纳指100、半导体等相关主题的最新研究或观点
- 野村 `Nomura` 关于标普500、纳指100、半导体等相关主题的最新研究或观点
- `Polymarket` 上与战争、衰退、降息、指数方向等相关的市场定价
- `WorldPEratio` 上相关指数的 `PE` 估值数据
- `danjuanfunds` 估值中心中的相关估值数据

如果用户讨论的是别的资产或主题，资料包应在上述默认清单基础上按需扩展。

## 资料包格式要求

会前资料包至少应包含：

- 来源名称
- 信息日期
- 核心结论或关键数字
- 对配置判断可能意味着什么

对外部信息要遵守以下规则：

- 同一份资料包必须发给全部 `5` 个专业角色
- 缺失的来源要明确说明，不要默默跳过
- 资料包只做事实整理，不提前替专业角色下结论
- 专业角色后续分析时，应把家庭约束与资料包信息一起纳入判断

## 独立上下文规则

每个角色 agent 都要被视为独立委员。

- 每个角色单独生成一个 agent
- 每个角色只能拿到：
  - 统一整理后的家庭投资案例输入
  - 同一份由 `munger-synthesis-chair` 整理的会前资料包
  - 该角色自己的提示词文件
- 首轮阶段，不要把某个角色的输出传给别的角色
- 首轮阶段，不要让角色之间直接辩论或相互修正
- 首轮结束后，只有 `munger-synthesis-chair` 可以接收所有专业角色的输出
- 续谈阶段，只有 `munger-synthesis-chair` 可以把追问摘要转发给相关专业角色
- 角色之间依然不能直接互相通信

## 强制实例化规则

- 必须实例化 `5` 个专业角色 agent，再实例化 `1` 个总结 agent。
- 不允许用一个主 agent 直接模拟多个角色并输出“多视角总结”来替代角色实例化。
- 这套 skill 默认就是为了适配并发上限 `6` 而设计的，不应再扩编。

这是强制规则。只要缺失角色实例化，这个 skill 就没有被正确执行。

这是这个 skill 最关键的约束。如果角色之间发生上下文串联，投委会系统的价值会明显下降。

## 执行流程

1. 把用户输入整理成标准家庭投资案例模板
2. 由 `munger-synthesis-chair` 先收集外部研究与估值信息，整理统一的会前资料包
3. 从 `references/roles/` 中逐个读取 `5` 个专业角色文件
4. 把“标准化家庭案例 + 统一资料包 + 对应角色文件”发送给 `5` 个专业角色 agent
5. 并行生成 `5` 个专业角色 agent
6. 收集每个专业角色的首轮原始输出，不要提前消除分歧
7. 将首轮原始输出统一交给 `munger-synthesis-chair`
8. 由总结 agent 输出：
   - 会前资料包摘要
   - 最终结论
   - 芒格式核心判断
   - 主要共识
   - 关键分歧
   - 最终行动建议
   - 风险提醒
   - 可续谈问题
9. 最终返回：
   - 总结 agent 的综合结论
   - 如有必要，再附上 `5` 个专业角色观点明细

## 资产打分

当用户要做“资产打分”或“月度投资”分配时，使用 [tactical-monthly-scoring.md](references/tactical-monthly-scoring.md)。

这个能力只覆盖四类资产：

- `标普500`
- `纳指100`
- `A股低波红利（512890）`
- `黄金`

执行时遵守以下规则：

- 必须先按参考文件中的规则分别给四类资产逐项打分
- 每类资产满分 `9` 分
- 必须返回每类资产的分项得分、总分、排名和本月动作
- 必须返回“本月战术仓投资百分比”，默认口径为“占当月可动用战术仓的比例”
- 如果用户想看金额，可以在百分比之外再换算金额
- 必须应用参考文件中的约束：
  - 纳指100 当月战术仓最高不超过 `20%`
  - 黄金长期不超过总资产 `5%`
  - 半导体不参与战术仓
- 如果触发暂停规则，则战术仓暂停，只保留底仓定投

如果缺少关键数据：

- 明确指出缺失项会影响打分质量
- 可以给出“暂定分”和“待补数据项”
- 不要在关键估值、趋势或回撤数据缺失时假装精确

默认输出至少包含：

- 四类资产月度打分表
- 排名结果
- 本月战术仓分配百分比
- 本月动作建议
- 是否触发暂停规则
- 如有必要，补充默认顺序与限制说明

## 用户续谈规则

- 如果用户继续追问某个未解决冲突，可以开启续谈轮。
- 续谈轮只召回与该冲突直接相关的专业角色和 `munger-synthesis-chair`。
- 不要因为一个局部分歧而重跑整个投委会。
- 续谈输出要明确说明：这是对先前某个分歧点的继续讨论，而不是重新生成整套结论。

## 输出规则

默认最终回答优先呈现总结 agent 的综合结论，包括：

- 会前资料包摘要
- 最终结论
- 芒格式核心判断
- 主要共识
- 关键分歧
- 最重要的决策依据
- 清晰的下一步动作
- 风险提醒
- 如有必要，保留分歧与可续谈议题

只有在以下情况，才附上完整角色明细：

- 用户明确要求查看所有角色观点
- 决策金额大或风险高
- 各角色之间存在实质性分歧

## 语气与边界

- 分析要冷静、具体、克制
- 不要假装能准确预测短期涨跌
- 优先考虑本金安全、流动性安全和长期纪律，而不是“聪明感”
- 这个流程用于辅助判断，不替代持牌金融、税务或法律意见
