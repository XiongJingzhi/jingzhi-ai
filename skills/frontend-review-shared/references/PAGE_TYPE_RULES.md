# 页面类型规则

## 支持的类型

- landing
- dashboard
- form
- settings
- detail
- list
- auth
- onboarding
- unknown

## 推断规则

如果没有明确给出页面类型，则根据以下线索推断最可能类型：
- 标题和首屏内容
- 是否存在图表、筛选、指标
- 输入控件的数量和性质
- 页面是围绕单一实体、多实体列表，还是营销叙事展开

推断出的页面类型必须明确标记为“推测”。

## 页面目标规则

在评分前，先用一句话总结页面目标，并尽量基于证据给出最可信的判断。

如果页面目标不清：
- 直接指出
- 将合并总分上限限制为 `70`
- 视情况判断是否升级为 blocker

## 各类型提醒

### landing
- 价值、证明、行动应形成清晰顺序

### dashboard
- 先结论，后工具

### form
- 完成路径应短而被引导

### settings
- 可预期性和信任感比花哨更重要

### detail
- 实体和状态应成为第一阅读重点

### list
- 扫描和比较应轻松

### auth
- 去除歧义和干扰

### onboarding
- 让用户理解投入成本和下一步
