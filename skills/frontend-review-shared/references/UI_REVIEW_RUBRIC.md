# UI 评审 Rubric

## 目的

当你评估页面的结构、信息层级、视觉一致性和页面类型匹配度时，使用这份 rubric。

目标不是奖励“新奇”或“花哨”，而是判断这个页面是否帮助用户快速理解：
- 这是什么页面
- 重点是什么
- 下一步该做什么

## 全局原则

- 结构优先于装饰
- 页面目标优先于单点美观
- 一条主线优先于多个抢注意力的焦点
- 证据优先于个人口味
- 大问题优先于细碎瑕疵

## 默认评分维度

### 结构清晰度：25

重点看：
- 头部、控制区、主内容区、次内容区是否划分清楚
- 相关模块是否被合理分组
- 布局是否降低了扫描成本
- 是否避免了功能堆砌

扣分点：
- 多个区域权重相近，用户不知道先看哪里
- 筛选或页面装饰挤压了核心内容
- 卡片很多，但顺序和关系不清楚

### 信息层级：20

重点看：
- 首屏是否存在明确第一视觉焦点
- 主要信息、次要信息、辅助信息是否分层清楚
- 标题、正文、指标、操作是否有合理权重

扣分点：
- 多个元素同时争抢第一注意力
- 重要信息被控件或装饰压住
- 用户需要自己“找重点”

### 交互可理解性：15

重点看：
- 主操作是否明确
- 控件与结果之间的关系是否直观
- 关键交互后是否有反馈

扣分点：
- 用户不知道应该先点什么
- 按钮层级混乱
- 可点击区域看起来像静态内容，或反过来

### 视觉一致性：15

重点看：
- 字号层级是否统一
- 间距节奏是否稳定
- 圆角、边框、阴影语言是否一致
- 色彩使用是否克制

扣分点：
- 组件风格混杂
- 视觉强调过多、噪音过强
- 局部样式“拼装感”明显

### 响应式适配：5

重点看：
- 小屏下信息层级是否仍成立
- 堆叠顺序是否自然
- 点击区域是否仍可用

扣分点：
- 桌面布局被强行压缩到移动端
- 溢出、裁切、错位、阅读顺序混乱

## 页面类型重点

### landing
- 价值表达必须在快速浏览中说清
- CTA 必须明确
- 信息节奏应从承诺到证明再到行动

### dashboard
- 用户应先看到结论，再看到复杂工具
- 指标、趋势、图表的角色要区分清楚
- 筛选应服务分析，而不是主导首屏

### form
- 完成路径应线性、顺畅
- 字段分组应有明确逻辑
- 视觉噪音不应增加认知负担

### settings
- 分组要稳定、可预期
- 危险操作必须隔离
- 如果设计感削弱了信任感，应扣分

### detail
- 实体本身与当前状态要尽快被识别
- 辅助信息应围绕实体展开，而不是和实体抢焦点

### list
- 行或卡片扫描应轻松
- 排序、筛选、批量操作不应压过主浏览任务

### auth
- 下一步必须一眼可知
- 不必要的视觉干扰应被扣分

### onboarding
- 每一步都应说明这一步为什么存在、接下来会发生什么
- 进度与投入成本应清楚

## 目标不清的分数上限

如果页面在首屏无法说清目标，合并后的总分不得超过 `70`。

## 证据规则

- 优先使用截图和 DOM 证据，而不是只看源码猜测
- 如果一个结论依赖缺失的交互证据，必须标记为推测
- 不要虚构未观察到的状态

## 输出要求

每个评分维度都应包含：
- 观察
- 主要问题
- 证据
- 修改建议

建议必须尽量具体，不能只写“优化层级”或“提升体验”。
