# 评审循环策略

## 目的

定义什么时候编码循环继续，什么时候可以停止。

## 默认 Score Gate

- `final_score >= 85`

这个阈值可以按任务调整，但 judge 必须明确报告当前采用的阈值。

## 默认 Quality Gate

以下条件必须全部满足：
- 没有 blocker
- 结构清晰度 `>= 18/25`
- 信息层级 `>= 14/20`
- 交互可理解性 `>= 10/15`
- 状态完整性 `>= 6/10`
- 响应式适配 `>= 3/5`
- 可访问性 `>= 3/5`
- 工程与性能卫生 `>= 3/5`

## 最终停止规则

只有同时满足以下条件，评审循环才停止：
- `score_gate_passed = true`
- `quality_gate_passed = true`

否则：
- `should_continue_iteration = true`

## Judge 行为规则

- 只要存在 blocker，就必须让 quality gate 失败
- 如果页面目标不清，总分上限不得超过 `70`
- 某 critic 明确指出的严重问题，不应因为另一个 critic 没提到就被忽略
- 不允许用视觉好看掩盖主流程损坏或无障碍失败

## 对主 Agent 的交接要求

judge 应以“最小但有推进作用”的修复集合结束本轮评审。优先给出高杠杆、可落地的修改，而不是泛泛的“大改版”建议。
